docker操作命令

发表于 2022-11-13 分类于 docker 本文字数： 1k 阅读时长 ≈ 4 分钟

镜像命令

查找网络镜像(search)

docker search <image-name>:<tag>

-a 选项显示本地镜像
-q 仅显示镜像ID

分页查找

docker search <image>:<tag> --limit 5

拉取镜像(pull)

docker pull <image>:<tag>

推送镜像

docker push <username>/<image>:<tag>

列出镜像(image)

查看本地所有镜像

docker images
docker image ls

删除镜像(rmi)

docker rmi <image>

强制删除

docker rmi <image> -f

删除全部

docker rmi -f ${docker images -qa}

虚浮镜像

虚浮镜像指的是只有镜像ID，没有镜像名的镜像

删除虚浮镜像

docker image prune

标记镜像

可以将一个镜像标记成一个新的镜像，新的镜像和原来的镜像是同一个镜像ID

docker tag <image>:tag <new-image>:tag

容器命令

运行容器(run)

以守护进程形式启动

docker run -d <image>
--name="容器名" # 系统默认随机
--restart=always # 自启动
-d  #以守护进程形式运行，返回容器id
-i  #交互式运行，通常与-t同时使用
-t  #为容器重新分配一个伪输入终端，通常与-i同时使用
-P # 随机端口映射 -p 6379:6379 宿主机port:容器port
-p # 指定端口映射

以交互式形式启动

docker run -it <image> /bin/bash

停止容器(stop)

docker stop <container>

强制停止

docker kill <container>

启动停止容器(start)

docker start <container>

重启容器(restart)

docker restart <container>

查看容器(ps)

查看正在运行的容器

docker ps

查看所有容器

docker ps -a
[options]
-a # 列出所有运行过的容器 正在运行 + 历史记录
-l # 列出最近创建的容器
-n # 显示最近n个创建的容器
-q # 静默模式，只显示容器编号

移除容器(rm)

docker rm <container>

强制移除

docker rm -f <container>

移除所有

docker rm -f ${docker ps -a -q}
docker ps -a -q | xargs docker rm

容器日志(logs)

docker logs  <container>

容器进程(top)

docker top <id> or <name>

容器详情(inspect)

docker inspect <id>

容器交互(exec)

exec命令

docker exec -it debian11 /bin/bash

attach命令

docker attach -it debian11 /bin/bash

区别:

attach直接进入容器启动命令的终端，不会启功新的进程，使用exit退出 会导致容器停止

exec 是在容器中打开行的终端，并且可以启动新的进程，使用exit退出不会导致容器停止

容器文件复制(cp)

docker cp debian11-1:/tmp/test.txt /tmp/test.txt

导出容器实例(export)

docker export <id> > filename.tar

导入容器实例(import)

car abcd.tar | docker import - qibria/ubuntu:3:7

提交容器实例生成镜像文件(commit)

docker commit -m="描述信息" -a="作者" <container-id> <image-name>:<tag>

容器自启动

docker run -d --restart=always --name <name> <image>
docker update --restart=always <container>

容器数据卷

docker volume

容器卷在宿主机上的路径

/var/lib/docker/volumes

创建数据卷

docker volume create <name>
docker volume create my-vol

查看数据卷

docker volume ls

查看指定 数据卷 的信息

docker volume inspect my-vol

删除数据卷

docker volume rm my-vol

移除空闲卷

docker volume prune

启动时挂载卷

docker run -d -P \
    --name web \
    # -v my-vol:/usr/share/nginx/html \
    --mount source=my-vol,target=/usr/share/nginx/html \
    nginx:alpine

也可以直接挂载的宿主机上面

要使用绝对路径

docker run -d -P \
    --name web \
    # -v my-vol:/usr/share/nginx/html \
    --mount source=/tmp/my-vol,target=/usr/share/nginx/html \
    nginx:alpine

数据卷权限

默认权限(rw)

docker run -it --privileged=true -v <source>:<target> <image>

显式指定

docker run -it --privileged=true -v <source>:<target>:rw <image>

rw 可读可写

r 只读

网络

网络模式

  • bridge：使用--network bridge指定，默认使用docker0

  • host：使用--network host指定

  • none：使用--network none指定

  • container：使用--network container：NAME或者容器ID指定

查看网络

docker network ls

详情

dokcer network inspect <network-name>

新建网络

docker network create aa_network

删除网络

docker network rm <network-name>

容器网络

docker run -d -p 8081:8080 --network my_network --name tomcat81

tips: 使用同一个自定义网络的容器可以通过容器名直接调用，可以ping <name> 连通

docker hub

登录

docker login

推送镜像至docker hub

docker push <username>/<image>:<tag>

windows 10 在 vscode 中配置 git bash 终端

发表于 2022-11-11 分类于 vscode ， git 本文字数： 144 阅读时长 ≈ 1 分钟

打开设置搜索Terminal Integrated window

找到settings.json文件打开

找到terminal.integrated.profiles.windows

"terminal.integrated.profiles.windows": {
    "PowerShell": {
        "source": "PowerShell",
        "icon": "terminal-powershell"
    },
    "Command Prompt": {
        "path": [
            "${env:windir}\\Sysnative\\cmd.exe",
            "${env:windir}\\System32\\cmd.exe"
        ],
        "args": [],
        "icon": "terminal-cmd"
    },
    "GitBash": {
        //"source": "Git Bash",
        "path": ["D:\\Program Files\\Git\\bin\\bash.exe"],
        "icon": "terminal-bash"
    }
},
"terminal.integrated.defaultProfile.windows": "GitBash",

添加Git Bash配置

"GitBash": {
   //"source": "Git Bash",
   "path": ["D:\\Program Files\\Git\\bin\\bash.exe"], 
   "icon": "terminal-bash"
}

path 的参数是git的安装路径下的/bin/bash.exe

注意GitBash可以是Git-Bash但不可以是Git Bash 配置文件中避免出现空格

这段配置将Git Bash设置为默认终端

"terminal.integrated.defaultProfile.windows": "GitBash",
